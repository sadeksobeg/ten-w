# تقرير تنفيذ موقع T.E.N.E.G.T.A — وخطة التطوير نحو تجربة عالمية

هذا الملف يجمع **(أ)** تقريراً تفصيلياً بما تم بناؤه فعلياً في المشروع، و**(ب)** خطة شاملة ومرحلية لرفع الموقع إلى مستوى احترافي عالمي مبهر.

**نطاق المشروع الحالي:** موقع تسويقي للشركة فقط (مجلد `site/`). تمت إزالة أي مسار سابق متعلق بنظام عيادات أو API داخلي ليتماشى المستودع مع هوية الموقع كواجهة شركة.

---

## الجزء أ — تقرير تفصيلي بما تم تنفيذه

### أ-1. المكدس التقني

| المكوّن | التفاصيل |
|---------|-----------|
| **الإطار** | Next.js 16 (App Router)، React 19، بناء إنتاجي عبر Webpack (`--webpack`) لتوافق بيئات Windows عند الحاجة. |
| **الأنماط** | Tailwind CSS v4 مع متغيرات ألوان وخطوط في `globals.css` و`@theme inline`. |
| **التعدد اللغوي** | `next-intl` v4؛ المسارات تحت `app/[locale]`؛ اللغات: العربية (`ar`) والإنجليزية (`en`) مع `localePrefix: "always"`؛ الافتراضي في التوجيه: العربية. |
| **الاتجاه (RTL/LTR)** | `dir` و`lang` على `<html>` في تخطيط اللغة؛ خطوط عربية (Cairo, Tajawal) ولاتينية (Inter). |
| **التحقق من البيانات** | `zod` في مسار API الاتصال وفي النماذج حيث ينطبق. |
| **الحركة والبصريات** | Framer Motion، GSAP (استيراد ديناميكي حيث يلزم)، Lenis للتمرير السلس، Lottie عبر `lottie-react` مع بيانات مدمجة في `data/hero-lottie.json`. |
| **Three.js / R3F** | `@react-three/fiber` و`three` مثبتان ويُفترض استخدامهما في مكوّنات مثل الـ Hero (حسب التصميم الحالي). |
| **CMS اختياري** | تكامل Strapi عبر `STRAPI_URL` و`STRAPI_API_TOKEN`؛ بدون CMS يُستخدم محتوى احتياطي من `lib/fallback-data.ts`. |

### أ-2. هيكل التطبيق والملفات الأساسية

| المسار / الملف | الوظيفة |
|-----------------|----------|
| `app/layout.tsx` | تمرير الجذر فقط؛ لا يحتوي `html/body` (يُعرّفان في تخطيط اللغة). |
| `app/[locale]/layout.tsx` | خطوط، `NextIntlClientProvider`، خلفية، مؤشر مخصص، تمرير سلس، رأس، محتوى رئيسي، تذييل، رابط «تخطي إلى المحتوى»، Google Analytics اختياري. |
| `middleware.ts` | توجيه i18n عبر `next-intl` (ملاحظة Next 16: توثيق الانتقال من `middleware` إلى `proxy` مستقبلاً). |
| `i18n/routing.ts` | تعريف اللغات والسلوك. |
| `i18n/request.ts` | تحميل الرسائل حسب اللغة. |
| `i18n/navigation.ts` | `Link`, `useRouter`, `usePathname` متوافقة مع التوجيه. |
| `messages/ar.json`, `messages/en.json` | كل نصوص الواجهة القابلة للترجمة تقريباً. |
| `lib/site.ts` | عنوان الموقع الكنسي (لـ metadata وsitemap). |
| `lib/metadata-helpers.ts` | مساعدات لعناوين الصفحات والوصف. |
| `lib/strapi.ts` + `lib/fallback-data.ts` | جلب المحتوى أو البدائل. |
| `app/api/contact/route.ts` | استقبال نموذج الاتصال: JSON، honeypot، إرسال اختياري لـ webhook. |
| `app/sitemap.ts`, `app/robots.ts` | SEO أساسي. |
| `next.config.ts` | `transpilePackages: ["three"]`, رؤوس أمان (`X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy`), أنماط صور عن بُعد لـ Strapi وlocalhost. |
| `.env.example` | توثيق المتغيرات العامة (الموقع، واتساب، Strapi، نموذج الاتصال، GA، الشبكات الاجتماعية). |
| `cms/README.md` | إرشادات حقول Strapi المقترحة. |

### أ-3. الصفحات والمسارات المنفذة (لكل من `/ar/...` و`/en/...`)

| المسار النسبي | المحتوى الوظيفي |
|---------------|-----------------|
| `/` | الصفحة الرئيسية: Hero، أقسام خدمات، مشاريع مميزة، دعوة للتواصل، بيانات منظمة JSON-LD للمنظمة. |
| `/about` | من نحن، رؤية وقيم، تمييز، خط زمني. |
| `/solutions` | حلول مقسمة (ذكاء اصطناعي، أمن، برمجيات، بنية تحتية، تحول رقمي) مع CTA. |
| `/contact` | نموذج اتصال، معلومات، رابط واتساب اختياري. |
| `/projects` | قائمة المشاريع. |
| `/projects/[slug]` | تفاصيل مشروع: تحدٍ، حل، نتائج. |
| `/pilots` | صفحة المختبرات/الطيارات التجريبية. |
| `/tech-stack` | عرض التقنيات والمنصة. |
| `/team` | أعضاء الفريق (بيانات احتياطية أو من CMS). |
| `/careers` | وظائف مع JSON-LD للوظائف حيث ينطبق. |
| `/investors` | محتوى للمستثمرين (قابل للتوسعة). |
| `/blog` | قائمة المقالات. |
| `/blog/[slug]` | مقال مع JSON-LD للمقال. |
| `/legal` | روابط للصفحات القانونية. |
| `/privacy`, `/terms` | نصوص قانونية أولية جاهزة للاستبدال عند الإطلاق الرسمي. |

معظم الصفحات تستخدم **`generateMetadata`** (عنوان، وصف، `alternates` للغات) لتحسين الظهور في محركات البحث والمشاركة الاجتماعية.

### أ-4. المكوّنات الرئيسية (مجلد `components/`)

- **التخطيط:** `Header` (تنقل متجاوب، قائمة جوال، `LocaleSwitcher`)، `Footer`، `AmbientBackground`، `CustomCursor`، `SmoothScroll`.
- **الرئيسية:** `Hero`، `HeroMotion`، `HeroDecor`، `HeroLottie`، أقسام الخدمات والمشاريع والـ CTA.
- **الاتصال:** `ContactForm` مع التحقق وتجربة المستخدم.
- **التحليلات:** `GoogleAnalytics` عند ضبط معرف القياس.
- **SEO منظم:** `OrganizationJsonLd` وأنواع أخرى حسب الصفحة.

### أ-5. الأمان والخصوصية (مستوى الموقع التسويقي)

- نموذج الاتصال: **honeypot** لتقليل السبام الآلي.
- رفض الطلبات المشبوهة من الـ API مع استجابة JSON واضحة.
- إمكانية إعادة توجيه الإرساليات إلى **webhook** خارجي (`CONTACT_WEBHOOK_URL`) دون تخزين إلزامي في قاعدة بيانات الموقع.
- رؤوس HTTP أساسية في `next.config.ts`.

### أ-6. إمكانية الوصول والأداء (ما هو منفّذ كمبدأ)

- دعم **`prefers-reduced-motion`** في CSS وبعض مسارات الرسوم.
- رابط تخطي للمحتوى في أعلى الصفحة.
- تركيز مرئي (`:focus-visible`) عبر المتغيرات العامة.

### أ-7. الجودة والبناء

- سكربت **`npm run test:a11y`** يشير إلى `scripts/a11y-note.mjs` كتذكير لتدقيق يدوي (Lighthouse / axe).
- **`npm run lint`** عبر ESLint مع إعداد Next.

### أ-8. ما تم استبعاده عن نطاق «موقع الشركة» (للتوثيق)

- لا يوجد في المستودع الحالي: خادم عيادات، Docker لقاعدة بيانات العيادة، n8n، صفحات تسجيل دخول/استقبال طبية، أو تبعيات مثل React Query / Sentry / Playwright مرتبطة بتلك التجربة. المشروع موحّد ك**موقع شركة T.E.N.E.G.T.A**.

### أ-9. توسعة المنصة كشركة ذكاء اصطناعي متعددة المجالات

الهوية المستهدفة هي **شركة ذكاء اصطناعي وهندسة أنظمة** تخدم **مجالات متعددة**، وليس تخصيصاً لقطاع واحد (مثل الزراعة فقط). أي مسودة سابقة ركّزت على مجال واحد تُعامل كـ **سيناريو ضمن محفظة أوسع**.

راجع ملف **[PLATFORM-EXPANSION-PLAN.md](./PLATFORM-EXPANSION-PLAN.md)** لمسارات مقترحة محايدة (`/solutions/intelligent-systems`، `/demo/explore`، فيديو عام، تنويع دراسات الحالة) وتوضيح التموضع للمستثمرين والزوار.

## الجزء ب — خطة شاملة لجعل الموقع «كاملاً» واحترافياً عالمياً مبهراً

الهدف: تجربة **موثوقة**، **سريعة**، **متعددة اللغات بجدية**، **قابلة للصيانة**، **جمالية مميزة**، ومتوافقة مع توقعات العلامات التقنية العالمية.

### المرحلة 1 — الأساس الاحترافي (الثقة والاستقرار)

1. **محتوى قانوني ونهائي:** مراجعة قانونية لـ `/privacy` و`/terms` وإضافة سياسة ملفات تعريف الارتباط إن لزم، وعنوان بيانات GDPR إن كان الجمهور أوروبياً.
2. **مراقبة الأعطال:** دمج Sentry (أو بديل) للواجهة مع DSN عبر البيئة، دون إثقال البناء.
3. **CI/CD:** GitHub Actions (أو Azure DevOps): `lint` + `build` على كل PR، وفحص TypeScript صارم.
4. **البيئات:** فصل `preview` / `production` على Vercel (أو المضيف) مع متغيرات بيئة موثقة.

### المرحلة 2 — الأداء والـ Core Web Vitals (الإبهار «الصامت»)

1. **Lighthouse كهدف رقمي:** LCP، CLS، INP — قياس على `/ar` و`/en` والصفحات الثقيلة (الرئيسية).
2. **الصور:** `next/image` لكل الأصول الثقيلة، أبعاد صريحة، صيغ حديثة (AVIF/WebP)، lazy load للأقسام تحت الطية.
3. **الخطوط:** `subset` محسّن، `display: swap`، تقليل أوزان الخطوط غير الضرورية.
4. **JavaScript:** تقسيم الكود (dynamic import) لمكوّنات R3F والـ Hero الثقيلة على الموبايل؛ تقليل الجسيمات/التعقيد على الشاشات الصغيرة.
5. **التخزين المؤقت:** رؤوس Cache مناسبة للأصول الثابتة؛ مراجعة `fetch` لـ Strapi مع `revalidate` أو ISR حسب سياسة المحتوى.

### المرحلة 3 — إمكانية الوصول WCAG 2.2 (عالمية حقيقية)

1. **تدقيق نظامي:** axe-core في CI أو Playwright + `@axe-core/playwright` على الصفحات الرئيسية.
2. **لوحة المفاتيح:** كل القوائم الجوالة والنماذج والـ modals قابلة للتنقل بالكامل بـ Tab وEscape.
3. **الشاشات القارئة:** نصوص بديلة للصور التوضيحية، تسميات ARIA للنماذج والأزرار الأيقونية.
4. **التباين:** التحقق من نسب التباين على الذهبي/الرمادي على الخلفية الداكنة.

### المرحلة 4 — SEO وقابلية الاكتشاف الدولي

1. **hreflang وكنسي:** التأكد من `alternates.languages` و`metadataBase` متسقة مع النطاق النهائي.
2. **Structured data:** توسيع JSON-LD (BreadcrumbList، WebSite مع SearchAction إن وُجد بحث، FAQ للصفحات المناسبة).
3. **ميتا لكل لغة:** أوصاف فريدة وليست ترجمة حرفية ركيكة؛ كلمات مفتاحية طبيعية لكل سوق.
4. **خريطة موقع ديناميكية:** تضمين مقالات/مشاريع من CMS عند التشغيل الفعلي لـ Strapi.

### المرحلة 5 — الأمان والنماذج (منع الإساءة)

1. **CAPTCHA بشرية خفيفة:** Cloudflare Turnstile أو hCaptcha على نموذج الاتصال.
2. **Rate limiting:** على `app/api/contact` (حسب المضيف: Edge middleware أو KV).
3. **التحقق من البريد:** تنسيق وسياسة طول الرسالة ومنع الحقول الخطرة.

### المرحلة 6 — المحتوى والهوية البصرية «المبهرة»

1. **دليل هوية:** ألوان، مسافات، نبرة الكتابة (عربي/إنجليزي)، قواعد للصور والفيديو.
2. **مكتبة مكوّنات UI:** أزرار، بطاقات، أقسام متكررة موحّدة لتقليل التشتت البصري.
3. **صور وفيديوهات احترافية:** تصوير منتج/فريق حقيقي بدل stock فقط؛ فيديو Hero قصير (مع خيار عدم التشغيل التلقائي).
4. **حالات خاصة:** صفحات 404 و500 مخصصة بلغتين، متسقة مع العلامة.

### المرحلة 7 — التوسع اللغوي والأسواق

1. **جاهزية الترجمة:** مصطلحات موحدة في `messages/*`؛ تجنب النصوص المضمنة في JSX.
2. **لغات إضافية** (حسب الاستراتيجية): تركيا، فرنسا، إلخ — عبر نفس بنية `next-intl`.
3. **تواريخ وأرقام:** `Intl` للتنسيق المحلي حسب `locale`.

### المرحلة 8 — ميزات «تجربة راقية» اختيارية

1. **PWA:** تثبيت خفيف، أيقونة، شاشة بداية (إن كان هناك حاجة عملية).
2. **وضع قراءة / تباين عالٍ** (اختياري في إعدادات المستخدم).
3. **مدونة فعلية:** جدولة، مؤلفون، صور بارزة، مشاركة اجتماعية Open Graph غنية.

### المرحلة 9 — القياس والتحسين المستمر

1. **أهداف في GA4:** تحويلات نموذج الاتصال، نقرات CTA الرئيسية، عمق التمرير.
2. **اختبارات A/B** (لاحقاً): عناوين Hero أو أزرار CTA عبر أدوات مثل Vercel Flags أو منصة خارجية.

### المرحلة 10 — جدول زمني مقترح (مرن)

| الفترة | التركيز |
|--------|---------|
| **0–2 أسبوع** | قانوني، Sentry، CI، Turnstile، تحسين صور/خطوط، Lighthouse أول مرة. |
| **2–6 أسابيع** | WCAG، SEO معمّق، توحيد المكوّنات، محتوى بصري جديد. |
| **6–12 أسبوعاً** | لغات إضافية، PWA اختياري، تكامل CMS كامل للمدونة والمشاريع، تجارب A/B. |

## خلاصة

- **ما تم تنفيذه:** موقع شركة متعدد اللغات، غني بالصفحات والـ SEO الأساسي والحركة والتصميم الداكن الذهبي، مع نموذج اتصال آمن نسبياً وتكامل Strapi اختياري.
- **ما يجعله «عالمياً مبهراً»:** ليس تأثيراً بصرياً فقط، بل **أداء قياسي**، **وصول للجميع**، **محتوى وقانون جاهزان للإنتاج**، **مراقبة واستمرارية في النشر**، وهوية بصرية ونسيجية متسقة عبر كل الصفحات.
